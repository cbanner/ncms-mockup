# NCMS Website Refresh — Project Plan

**Version:** 1.0 | **Date:** March 2026
**Board Sponsor:** Chris Banner
**Status:** Draft for Review

---

## Purpose

This plan provides a structured path from the strategic mockups and recommendations already developed to a launched, optimized website for Nashua Community Music School. It assumes the mockups represent the *opportunity and direction* — not necessarily the final design or technology — and that key decisions remain open on platform, design approach, and implementation partner.

## Phase 1: Discovery & Requirements

**Goal:** Understand what the new site needs to do, what content exists, and what gaps need filling.

### Activities

- [ ] **Content audit** — Inventory every page on the current Squarespace site. For each: keep as-is, revise, merge, or remove
- [ ] **Stakeholder interviews** (1-on-1, 30 min each):
  - Lindsay: Organizational priorities, enrollment goals, fundraising targets
  - Cody: Registration workflow pain points, program details, event management needs
  - Brittany: Brand standards, content calendar, email/social integration, analytics wishlist
  - 1-2 board members: Donor perspective, community perception
- [ ] **Audience definition** — Refine the engagement hierarchy (from index4 mockup) into actionable personas:
  - Prospective student/parent (enrollment path)
  - Current student/family (retention, upsell)
  - Donor/supporter (giving path)
  - Community partner (collaboration)
  - General community member (awareness)
- [ ] **Functional requirements** — Document must-have vs. nice-to-have capabilities:
  - Online registration and payment
  - Event calendar and RSVP
  - Donation processing (one-time, recurring, campaigns)
  - Newsletter signup and email integration
  - Multilingual support (Spanish priority)
  - Analytics and conversion tracking
  - Blog/news capability
  - Faculty/staff directory
  - Scholarship application workflow
- [ ] **Competitive/peer review** — Look at 3-5 comparable community music school sites for inspiration and benchmarking

### Checkpoint: Requirements Review

**Present to committee:** Requirements document, content audit summary, audience personas
**Decision needed:** Sign-off on scope and requirements before technology evaluation

## Phase 2: Technology Evaluation

**Goal:** Decide whether to stay on Squarespace or move to another platform, based on requirements.

### Evaluation Criteria

| Criteria | Weight | Notes |
|----------|--------|-------|
| Staff maintainability | High | Brittany and Cody need to update content without developer help |
| Registration/payment integration | High | Online registration is a strategic priority |
| Cost (platform + ongoing) | High | Nonprofit budget considerations |
| Design flexibility | Medium | Ability to implement the design vision from mockups |
| SEO/schema markup support | Medium | AEO/GEO strategy requires structured data |
| Email/CRM integration | Medium | Mailchimp, Constant Contact, or similar |
| Multilingual support | Medium | Spanish content is a Bridge Communities priority |
| Donation platform integration | Medium | Giving pages, recurring gifts, campaigns |
| Migration complexity | Lower | One-time cost but worth considering |

### Platforms to Evaluate

- **Squarespace** (current) — Familiar, low cost, limited integration depth
- **WordPress + managed hosting** — Maximum flexibility, larger ecosystem, higher maintenance
- **Wix** — Similar to Squarespace, different tradeoffs
- **Specialized nonprofit platforms** (e.g., Morweb, Firespring) — Built-in donation/CRM features
- **Hybrid approach** — Squarespace for site + external tools for registration/donations

### Activities

- [ ] Score each platform against criteria (spreadsheet)
- [ ] Get cost estimates (platform fees, implementation, annual maintenance)
- [ ] If considering an agency: issue brief RFP or request proposals from 2-3 firms
- [ ] Assess migration effort from current Squarespace site

### Checkpoint: Platform Decision

**Present to committee:** Platform comparison matrix, cost analysis, recommendation
**Decision needed:** Platform selection and (if applicable) agency/partner selection

## Phase 3: Design & Information Architecture

**Goal:** Define site structure, page layouts, and visual design — building on the mockup direction but through a proper design process.

### Information Architecture

- [ ] **Sitemap** — Define full page hierarchy based on requirements and content audit
  - Primary navigation, footer navigation, utility navigation (language, login, search)
- [ ] **User flows** — Diagram the paths key personas take to accomplish their goals
  - Parent enrolling a child, donor making a gift, visitor exploring scholarships

### Design Progression

- [ ] **Mood boards** — 2-3 visual directions capturing the desired tone and feel
  - Colors, textures, typography samples, image treatments, reference sites
- [ ] **Style tiles** — Refined visual vocabulary without page layout
  - Font pairings, color palette, button styles, image treatment, iconography direction
- [ ] **Wireframes** — Low-fidelity page layouts showing content placement and navigation
  - Homepage, program detail, new student, donation, scholarship, events, about/faculty, contact
- [ ] **Design comps** — Full-color, high-fidelity mockups for key page templates
  - 2-3 rounds of revision, mobile and tablet responsive variations

### Design System & Accessibility

- [ ] **Design system / style guide** — Documented reference for all design components
  - Typography scale, color usage rules, button variants, spacing, card patterns, form styles
  - Ensures consistency during build and future staff-led maintenance
- [ ] **Accessibility review** — WCAG 2.2 AA compliance: color contrast (4.5:1 minimum), font sizes, keyboard navigation, heading hierarchy

### Checkpoint: Design Approval

**Present to committee + staff:** Sitemap, user flows, wireframes, high-fidelity page designs, style guide
**Decision needed:** Approve design direction before build begins

## Phase 4: Content Development

**Goal:** Create and collect all content for the new site. This is typically the longest pole — start early and run in parallel with design and build.

### Content Strategy

- [ ] **Voice, tone, and messaging framework**
  - Align with engagement hierarchy (who are we speaking to on each page?)
  - Define CTAs by audience segment
- [ ] **Content matrix** — Spreadsheet blueprint for the new site, mapping each page to:
  - Primary audience and goal, SEO target keyword, meta title and description
  - Content outline and required sections, calls to action, media needs
  - Content owner, status (draft / in review / approved), source (existing / needs writing)
- [ ] **Storytelling framework** — Templates for recurring content types
  - Student/family impact stories, instructor profiles, community impact narratives
  - Program descriptions focused on outcomes and experience, not just logistics

### Copywriting

- [ ] Homepage hero messaging and impact metrics
- [ ] Program descriptions, donation page copy and giving levels
- [ ] Student/family testimonials and scholarship narratives
- [ ] FAQ content (aligned with schema markup strategy)

### Photography & Media

- [ ] **Photo audit** — Inventory existing library for quality, coverage, and usage rights
- [ ] **Shot list and photo sessions** — 2-3 dedicated sessions capturing authentic moments
  - Individual lessons, group classes, recitals/performances, the facility, instructor portraits
- [ ] **Video planning** — Identify 2-3 key opportunities (student testimonials, class previews, virtual tour)
- [ ] **Photo consent forms** — Collect signed releases, especially for images of minors

### Multilingual & SEO

- [ ] **Multilingual content** — Translate priority pages into Spanish
  - Determine scope: full site vs. key pages (enrollment, scholarships, contact)
- [ ] **Schema markup preparation** — Finalize JSON-LD content using the existing implementation guide
- [ ] **Content migration plan** — Map existing content to new IA structure; identify what transfers, what needs rewriting, and gaps requiring new content

### Checkpoint: Content Review

**Present to committee + staff:** Content matrix, draft copy for key pages, media plan
**Decision needed:** Content approval (may require multiple rounds for key pages)

## Phase 5: Build & Integration

**Goal:** Build the site on the chosen platform, integrate third-party tools, and set up analytics and marketing foundations.

### Platform & Development

- [ ] Platform setup — Configure hosting, domain, SSL
- [ ] Template/page development — Build all page templates per approved designs and style guide
- [ ] Content population — Load all approved content, images, and media per migration plan
- [ ] Staff CMS training — Ensure Brittany and Cody can edit pages, add events, and publish posts

### Integrations

- [ ] Registration/enrollment system and payment processing
- [ ] Donation platform integration (one-time, recurring, campaigns)
- [ ] Email marketing (newsletter signup forms, list segmentation, welcome series automation)
- [ ] Event management (calendar, RSVP)
- [ ] Social media feeds and links

### Analytics & Tracking

- [ ] **Google Tag Manager** — Central container for all tracking tags
  - GA4 property tag, conversion event tags, remarketing tags, Meta Pixel
- [ ] **Google Analytics 4 (GA4)** — Pageviews, sessions, user demographics, audience segments
- [ ] **Conversion tracking** — Both macro and micro conversions
  - Macro: completed donation, completed registration, contact form submission
  - Micro: visited donate page, clicked "learn more" on a program, watched a video
- [ ] Google Search Console — Search performance, indexing, crawl errors
- [ ] Schema markup deployment (per implementation guide)
- [ ] Google Business Profile — Verify and optimize

### Digital Marketing Foundation

- [ ] UTM parameter strategy for campaign tracking
- [ ] Social sharing metadata (Open Graph tags) — Ensure shared links display correctly
- [ ] Meta Pixel — Install from day one to build audience data for future campaigns
- [ ] Google Ads remarketing tag — Build visitor audiences for Ad Grants campaigns
- [ ] SEO migration plan — Crawl old site, map all URLs, implement 301 redirects, submit new sitemap
- [ ] Google Ad Grants application — Apply during build so account is ready at launch

### Checkpoint: Staging Review

**Present to committee + staff:** Full site in staging environment, all integrations functional
**Decision needed:** Approval to proceed to testing and launch

## Phase 6: Testing & Launch

**Goal:** Ensure everything works, then go live.

### Functional Testing

- [ ] All forms submit correctly and send notifications to the right recipients
- [ ] Donation forms process test transactions successfully
- [ ] Registration/enrollment links connect properly
- [ ] Email signup forms add subscribers to the correct list
- [ ] All internal and external links work (no 404 errors)
- [ ] Media embeds (video, audio, maps, calendar) load properly

### Cross-Browser, Performance & Accessibility

- [ ] Cross-browser testing — Chrome, Safari, Firefox, Edge
- [ ] Mobile device testing — iOS and Android at multiple screen sizes
- [ ] Page speed testing — Target under 3 seconds (Google PageSpeed Insights)
- [ ] Image optimization — Compressed and properly sized
- [ ] **Accessibility testing**
  - Automated scans (WAVE, Lighthouse), keyboard navigation, screen reader testing
  - Color contrast verification, form label verification, heading hierarchy check

### Content QA

- [ ] Proofread every page for spelling, grammar, and factual accuracy
- [ ] Verify all images have alt text
- [ ] Check phone numbers, addresses, and email addresses
- [ ] Verify PDFs and downloadable files open correctly

### User Acceptance Testing

- [ ] New student finds and starts registration
- [ ] Donor makes a gift
- [ ] Parent finds lesson schedule and teacher info
- [ ] Visitor reads about scholarships and applies
- [ ] Brittany updates a page, adds an event, publishes a blog post

### Launch Checklist

- [ ] All 301 redirects in place and tested
- [ ] robots.txt allows crawling, XML sitemap submitted to Search Console
- [ ] GA4 and Google Tag Manager verified and collecting data
- [ ] SSL certificate active (HTTPS on all pages)
- [ ] Favicon, Open Graph images, and social sharing metadata configured
- [ ] Custom 404 page created
- [ ] Privacy policy page in place
- [ ] Update external profiles — Social media, directory listings, Google Business Profile, GuideStar

### Launch Sequence

- [ ] **Soft launch** — Site live but not yet promoted; monitor for errors and analytics issues
- [ ] **Full launch** — Announce via email to current families and donors, social media, board communication

### Checkpoint: Launch Approval

**Present to committee:** QA results, launch checklist status, any outstanding issues
**Decision needed:** Go/no-go for full launch

---

## Phase 7: Post-Launch Optimization

**Goal:** The site is live — now make it work harder over time.

### First Month: Stabilization

- [ ] Weekly monitoring — Broken links, 404 errors, redirect issues, crawl errors in Search Console
- [ ] Verify analytics data is collecting accurately
- [ ] Fix any issues surfaced by real-world traffic patterns

### Ongoing Reporting Cadence

- [ ] **Monthly analytics review** (Brittany + Chris)
  - Traffic trends, top pages, referral sources, new vs. returning visitors
  - Conversion rates: registration, donation, newsletter signup, contact form
  - Search performance: keyword rankings, organic impressions, click-through rates
- [ ] **Quarterly board report** — Key website metrics tied to strategic plan KPIs
- [ ] **Annual site review** — Comprehensive assessment, technology check, strategic alignment

### Content & SEO Maintenance

- [ ] Content freshness — Event calendar (ongoing), student stories (quarterly), faculty updates, blog posts (monthly target)
- [ ] SEO refinement — Optimize underperforming pages, build internal linking, pursue local SEO opportunities
- [ ] Broken link checks — Monthly scan for broken internal and external links
- [ ] Accessibility maintenance — Quarterly re-scan, fix issues as they arise

### Website Governance

- [ ] **Roles** — Define who can edit what, approval flow for publishing content
- [ ] **Editorial calendar** — Planned content for the next 3-6 months
- [ ] **Update schedule** — Defined cadence for when different sections get refreshed

### Digital Marketing Activation

- [ ] **Google Ad Grants** ($10K/month free for nonprofits) — Search ads for enrollment, events, scholarships
  - Must maintain 5% CTR, minimum 2 ad groups per campaign, active conversion tracking
- [ ] **Email marketing** — Segmented campaigns by audience and engagement level, welcome series for new subscribers
- [ ] **Retargeting** — Re-engage site visitors who didn't convert via Google Ads and Meta audiences
- [ ] **Social media advertising** — Targeted campaigns for enrollment periods, fundraising drives
- [ ] **A/B testing** — Homepage messaging, CTA placement, donation page giving levels, email subject lines
